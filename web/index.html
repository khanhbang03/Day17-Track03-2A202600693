01

Vì sao cần cả Baseline Agent và Advanced Agent?

+

Baseline là nhóm đối chứng. Chạy cùng input giúp chứng minh recall tăng đến từ User.md, còn giảm context cost đến từ compaction—thay vì chỉ khẳng định Advanced “tốt hơn”.

CONTROL GROUP
02

Ba lớp memory giải quyết ba vấn đề nào?

+

Short-term giữ follow-up chính xác; User.md giữ facts qua session; compact memory nén lịch sử cũ. Mỗi lớp có lifecycle riêng nên không phải một storage duy nhất gánh mọi loại dữ liệu.

SEPARATION OF CONCERNS
03

Benchmark có công bằng giữa hai agent không?

+

Có ở cấp input: cùng dataset, thứ tự turn, recall questions và token estimator. Offline deterministic loại bỏ sampling noise. Giới hạn là quality/token vẫn dùng heuristic, chưa phải tokenizer và LLM judge production.

FAIR, WITH LIMITS
04

Vì sao Advanced đắt hơn ở Standard Benchmark?

+

Conversation ngắn chưa kích hoạt compaction, nhưng Advanced vẫn nạp profile mỗi lượt. Vì vậy prompt cost là 23,699 so với 12,852. Persistent memory có overhead cố định; compact không phải lúc nào cũng thắng.

EXPECTED OVERHEAD
05

Vì sao Advanced thắng trong stress test?

+

Baseline kéo toàn bộ lịch sử lên 21,718 prompt tokens. Advanced compact bảy lần, chỉ xử lý 15,362 tokens—giảm 29.3%—mà recall vẫn đạt 1.000.

MEASURED BENEFIT
06

Vì sao phải đo Prompt tokens processed?

+

Compaction giảm context đầu vào phải đọc lại, không trực tiếp làm output ngắn hơn. Vì thế prompt tokens processed phản ánh đúng lợi ích hệ thống hơn chỉ số Agent tokens only.

RIGHT METRIC
07

Correction và conflict được xử lý thế nào?

+

Facts được lưu theo field. upsert_fact() giữ một giá trị hiện hành cho mỗi key, nên backend engineer bị thay bởi MLOps engineer, không tồn tại đồng thời.

ONE CURRENT TRUTH
08

Hệ thống ngăn false memory ra sao?

+

Chỉ explicit assertions khớp pattern mới được ghi. Câu hỏi, joke, phủ định và travel tạm thời bị bỏ qua. Điểm yếu: regex bảo thủ có thể bỏ sót cách diễn đạt hợp lệ.

CONFIDENCE FILTER
09

Rủi ro lớn nhất của Advanced Agent là gì?

+

False persistence, privacy và information loss khi summary. Agent nhớ lâu hơn đồng nghĩa fact sai cũng nguy hiểm hơn, profile cần governance hơn, và nén có thể mất chi tiết.

CAPABILITY ↔ LIABILITY
10

Nếu productionize, nên cải tiến gì trước?

+

Thêm provenance, timestamp, confidence, versioning, user controls, tokenizer thật, semantic extraction, encryption và retention policy. Đây là bước biến một memory demo tốt thành hệ thống có thể vận hành.
